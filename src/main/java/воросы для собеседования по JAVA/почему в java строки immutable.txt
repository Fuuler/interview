1) Безопасность.
String широко используется, как параметр для многих классов Java,
в частности для открытия сетевых соединений, подключений к БД,
открытию файлов и пр. И если бы строка изменялась,
то мы могли получить доступ к объекту (файлу например),
на который мы имеем право, затем изменить строку с именем (случайно или намеренно)
и получить доступ уже к другому файлу.
Так же String используется в механизме загрузки файлов,
и это -- фундаментальный аспект.
И если бы строка изменялась, то запрос на загрузку "java.io.Writer"
мог бы быть изменён на "DiskErasingWriter".

2) Hashcode
Из-за того, что строка не изменяется, она кэширует свой хэшкод
и не вычисляет его каждый раз, когда мы его вызываем,
что делает строку очень быстрой как ключ для hashmap.

3) Многопоточность
immutable делает экземпляры строк thread-safe.
